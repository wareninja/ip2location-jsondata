A simple & readable Java code, reusable to generate JSON data files from IP2Location Database (DBx.csv) files!
This library also makes an easy to use executable JAR, so you don't have to deal with the codebase.

How to use this library?

Option.1: use only the executable JAR!
Just download the latest jar from releases
Then you can run that jar file/lib from the command line. For example; 


$ java -jar ip2location-jsondata-0.0.1.jar data-folder-is-here/ IP2LOCATION-LITE-DB11.CSV

Remember that, you must have JVM/JDK in your local environment to be able to run this jar!

Option.2: use the source code!
Clone the project source code from github, then import as maven project in your ide!
Then; 

$ cd your_repo_base/repo_name
$ mvn clean install
this will generate the snapshot of executable JAR, which you can run as described in the prev option above.
...whenever in doubt, read the source luke!

What about the generated JSON data files?

Each generated file contains JSON representation of IP records, one per line.
A sample JSON record generated from IP2LOCATION-LITE-DB11.CSV; 



{"country_code":"JP","country_name":"Japan","region_name":"Tokyo","city_name":"Tokyo","zip_code":"214-0021","time_zone":"+09:00","_id":16797696,"ip_to":16798719,"ipaddress_from":"1.0.80.0","ipaddress_to":"1.0.83.255","_created":1424270870645,"createdAt":"2015-02-18T14:47:50+0000","lat":35.689506,"lng":139.6917,"latlng":[35.689506,139.6917]}
note that; "_id" field holds the data for "ip_from"!
The reason of naming it as "_id", instead of "ip_from", is to optomise the collection in MongoDB!
This, "_id", automatically generates an index when imported to MongoDB, and ensures uniqueness of imported IP2Location records!

These JSON data files can be used for variety of purposes.
The most useful of these is to store them in MongoDB, so to leverage the power of MongoDB search from your applications!
And you can easily import these JSON files into MongoDB with mongoimport
For example; 


mongoimport -h mongo_db_url:port -d db_name -c collection_name -u db_username -p db_pwd --upsert --file IP2LOCATION-LITE-DB11-1.json
